# Практическая работа №5. Наследование и полиморфизм

## Задание

В работе необходимо реализовать иерархию классов для заданного базового типа.

## Выбор варианта базового класса

В качестве базового класса можно использовать вариант из таблицы, либо предложить свой собственный базовый класс и
иерархию наследников (но отсутствующие в таблице).

Вариант для таблицы вычисляем по формуле:
`V = (10 * int(c1) + int(c2)) % 11`,
где `с1` и `c2` – первые буквы в фамилии и имени на английском языке в **верхнем регистре**.

## Требования к реализации

### Иерархия классов

* Помимо базового класса необходимо реализовать **не менее 5 производных классов** и по крайней мере **3 уровня в
  иерархии** (например, класс `С` наследует `B`, а класс `B` наследует `A`).
* В иерархии необходимо ввести **по крайней мере один абстрактный класс**.
* Задействовать **закрытое наследование** для одного классов.

### Структура проекта

Структурировать программу следующим образом:

* Каждый класс определяется двумя файлами (например, `Notebook.h` и `Notebook.cpp`).
* Файл `main.cpp` содержит код, связанный с консольным меню.
* Допускаются дополнительные файлы для структурирования кода.

### Требования к классам

* В каждом классе ввести **закрытые поля** и **открытые методы доступа** к полям.
* По крайней мере для одного из классов **устраните возможность создания объектов** с помощью конструктора без
  параметров и конструктора копирования.
* В базовом классе ввести **виртуальный метод `toString`** для получения информации об объекте в формате строки, в
  производных классах ввести переопределение виртуального метода.
* Ввести **статические элементы** (функции и неконстантные поля) хотя бы в **3 классах**.

### Основная программа (`main.cpp`)

* Ввести **динамический массив** для возможности работы с объектами разных типов. Число элементов определяется
  динамически при взаимодействии с пользователем.
* Для работы с объектами реализовать **консольное меню**.

## Функциональность консольного меню

Предоставить пользователю возможность **ввода объектов произвольного типа** и заполнения массива.

После ввода данных пользователь может получить следующую информацию, которая определяется по заполненному массиву:

1. **Количество объектов каждого типа** (без учёта наследования и возможности преобразования; например, `Animal = 0`,
   `Mammal = 1`, `Tiger = 2`, `Elephant = 3`).
2. **Количество объектов заданного типа** (с учётом наследования; например, для `Mammal` получаем 6 (с учётом `Tiger`,
   `Elephant`)).
3. **Информация по всем объектам**, а также по **выбранному объекту** (с помощью `toString`).
4. **Найти объекты, удовлетворяющие условию** по какой-либо характеристике, которая определена для **всех типов** в
   иерархии.
5. **Найти объекты, удовлетворяющие условию** по какой-либо характеристике, которая определена **только для некоторых
   типов** в иерархии.
